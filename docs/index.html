Install Guide

Simple setup for AMD and NVIDIA users

The Windows launcher is built to support both GPU families. On first run it detects the hardware path and installs the matching environment.

What normal users do

  1. Download the installer or portable zip.
  2. Start the app.
  3. Wait for first setup to finish.
  4. The launcher picks the correct GPU path automatically.

GPU support

  • AMD Radeon: supported
  • NVIDIA: beta support right now
  • Windows: primary target
  • First run: internet needed for bootstrap setup

Minimum I currently suggest

  • VRAM: 8 GB
  • RAM: 16 GB
  • CPU: 6-core

This is the lowest setup I currently suggest. It may still be slow, unstable, or give weaker results on this level, especially on harder subjects.

NVIDIA test note

NVIDIA support is still in a beta phase. Main development was done on AMD hardware, so I would really appreciate real NVIDIA feedback.

If you test it and hit issues, have ideas, or want to help, please email yanis.thispc.email@gmail.com.

Release Link Setup

How to make the download buttons work for real

This part is simple: upload your files somewhere public, then paste those public links into website/app.js.

Fields you replace

  • repo = your GitHub repo URL
  • releases = your GitHub Releases page
  • downloadInstaller = direct public installer URL
  • downloadPortable = direct public portable zip URL
  • issues = your bug report page
  • donate = your Ko-fi page

Easy order

  1. Upload the installer and zip to GitHub Releases.
  2. Copy the public file URLs.
  3. Open website/app.js.
  4. Paste the real URLs into PUBLIC_LINKS.
  5. Upload the website folder.

No-problem version

If you forget what goes where, open the website readme. I added the exact file names and a copy-paste example for the installer and portable download links.

Submission Flow

How the site should behave in v1

The main goal is trust and clarity, not instant chaos.

01

Official Releases

Only the owner account can upload app installers, portable zips, screenshots, and changelog text.

02

User Plugin Upload

Registered users upload a plugin zip, version, tags, screenshots, short summary, and compatibility notes.

03

Pending Review

New community uploads stay hidden until you approve, reject, or send them back for fixes.

04

Public Plugin Page

Approved plugins get a clean public page with download links, version history, screenshots, and author info.
