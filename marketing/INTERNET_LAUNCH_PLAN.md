# Internet Launch Plan for 3DVisual Mesh

## Best choice

Do an organic launch first, not paid ads.

Use this order:

1. GitHub repo
2. GitHub release
3. Live website
4. Product Hunt
5. Show HN
6. Blender and game-dev communities
7. Short demo posts on social platforms

## Why

- The app is still beta and needs feedback more than ad spend.
- Organic launch gives real bug reports, real hardware feedback, and better trust.
- Paid ads are easy to waste when onboarding and results still need polish.

## Best public message

Be honest:

- local image-to-mesh helper for Windows
- AMD-first workflow
- NVIDIA path included, but still beta-tested compared with AMD
- mesh is not guaranteed to be 100 percent production-finished
- light Blender cleanup may still be needed
- can also be useful as a starting point for 3D printing

That honesty will help more than hype.

## Launch order I recommend

### 1. Core pages first

You already have the main three:

- repo
- release
- website

That part is ready.

### 2. Product Hunt

Use Product Hunt for discovery, screenshots, and launch-day traffic.

Best angle:

- open-source
- local workflow
- Windows-friendly
- AMD-friendly
- Blender/game-dev helper

### 3. Show HN

Use Show HN if you want technical feedback and blunt opinions.

Best angle:

- local image-to-3D mesh workflow
- Windows + AMD focus
- open-source beta
- honest limits

### 4. Blender and game-dev communities

Good places to share:

- Blender Artists
- Blender Discord servers
- indie game dev Discord servers
- Unreal / Unity art communities

Share screenshots, one short demo video, and one honest sentence about current limits.

### 5. Reddit only where rules allow

Do not spam broad subreddits.

Post only where:

- self-promo is allowed
- you are showing real value
- you include before/after screenshots
- you ask for testing feedback

## What to avoid

- broad paid Google ads right now
- broad Meta ads right now
- pretending the mesh is game-ready in every case
- hiding that cleanup is sometimes needed
- overpromising NVIDIA maturity if you mainly tested on AMD

## Best sponsor path

### Best choice

Use `Ko-fi` now.

### Why

- fastest to understand
- simple for one-time support
- already linked in your project

### Second best

Enable `GitHub Sponsors` later for recurring support if you want long-term open-source funding.

### Real downside

GitHub Sponsors setup needs your own payment and tax details, so only you can finish that part.

## What people should do when they test

Ask testers to report:

- GPU model
- VRAM
- RAM
- Windows version
- AMD or NVIDIA
- subject type
- screenshots of result
- what failed

## Launch checklist

- add 3 to 5 clean screenshots
- add 1 short video or GIF
- pin the website link in the repo
- pin the release link in the website
- ask users to open issues with hardware details
- mention Ko-fi support as optional
